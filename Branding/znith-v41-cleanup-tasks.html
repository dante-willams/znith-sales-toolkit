01Tokens · Extend the palette

CLN41-01Add Info role + semantic aliases to shared tokensP0
Effort: 10 min · Single file · Zero breaking changes
File: shared/znith-tokens.css
Why: The v4 palette has only one accent role (amber) and amber is being used for eyebrows, stage-play callouts, section labels, and CTAs interchangeably. Adding an Info role (muted slate) gives tools a neutral-but-distinct color for non-status informational content, which lets amber reclaim its "Action" meaning.

--info-faint:   rgba(122,148,184,0.06);

/* Semantic role aliases — keeps old tokens alive, introduces role vocabulary */

--accent-glow:  var(--amber-glow);
--accent-muted: var(--amber-muted);

02Eyebrows · Demote from amber to neutral

CLN41-02Make section eyebrows mono-gray, not amberP0
Effort: 20 min · Global find-replace pattern
Files: dashboard, win-room, sales-hub, prospecting-agent, account-brief-v2, value-map (skip qualify-iq, demo-brief)
Why: Every tool has 1–4 rules like .section-eyebrow { color: var(--amber) }. That's amber on labels that repeat 5–10× per screen, which is what makes amber feel like skin instead of signal. Switching eyebrows to mono-gray instantly reclaims amber's meaning.
/* Find all rules matching this shape */
.section-eyebrow,
.eyebrow,
.side-label {
  color: var(--amber);
}

/* Replace with */
.section-eyebrow,
.eyebrow,
.side-label {
  font-family: var(--f-mono);
  font-size: 10px;
  font-weight: 700;
  letter-spacing: 0.14em;
  text-transform: uppercase;
  color: var(--tfaint);
}

Grep pattern for Claude Code: (eyebrow|eye|section-label|side-label)[\s\S]{0,200}color:\s*var\(--amber

09Migration strategy · How hard is this really?

You asked directly — here's the honest read:

What makes this easy

  • You already did the heavy lift. Shared tokens file, Conga chrome, /Znith footer, theme toggle, Conga Symbol SVG — all landed on brand-refresh. None of that changes.
  • No breaking changes. All existing var(--amber*) references continue to work. You only replace them where meaning is wrong, one section at a time.
  • Central leverage. Tasks 01 (add tokens) and 02 (demote eyebrows) together hit ~70% of the surface with one shared token update + one grep-replace. That's an hour of work.
  • Two tools are already close. qualify-iq (11 refs) and demo-brief (9 refs) likely need zero work beyond the shared-token update.

What makes this a real project, not a patch

  • Prospecting Agent is 5× heavier than anything else. 163 amber references means amber is baked into the structure, not sprinkled on top. Budget 2 hours, expect to find ~30 edits.
  • Per-tool taste calls. "Which metric is the hero?" and "is this button action or navigation?" aren't mechanical find-replace — they need a designer's eye. But they're 10-min decisions, not hours.
  • Theme parity. Every change has to look right in both dark and light. Existing shared tokens handle this automatically as long as you only use role tokens (--info, --accent, etc.), never raw hex.

10What NOT to do

DONT-01Do not remove --amber tokensP2

Keep --amber, --amber-deep, --amber-muted, --amber-faint, --amber-glow forever. They're the physical color. --accent is just a semantic alias. Removing them breaks 163 references in one file alone.

DONT-02Do not do a big-bang rewriteP2

Ship foundation (sprint 1+2) first, let it bake, then tackle prospecting-agent. Each PR should be visually coherent on its own — reviewer should be able to open the diff and see "eyebrows changed" as one clear intent, not 40 scattered edits.

DONT-03Do not touch favicons, logos, or chromeP2

v4.1 is a palette-meaning change, not a brand refresh. Conga Symbol stays. /Znith footer stays. Theme toggle stays. Amber hue stays (the color is right; only its role changes).
