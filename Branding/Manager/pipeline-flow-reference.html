03 · Spec
Container
.stage-card — surface panel, --clip-lg chamfer, 22px padding, --sh-sm shadow
Grid
.stage-flow — 5 equal columns, 8px gap. Add .four for 4-column variant.
Cell
.stage-block — directional chamfer via --clip-fwd. Left accent stripe via ::before.
States
.done — green stripe, green label, subtle green wash
.active — amber stripe, amber label, --amber-wash background
(none) — --border-md stripe, faint label
Cell content
.stage-lbl — mono 8px eyebrow
.stage-name — Epilogue 13px/700 stage name
.stage-val — mono 10px "value · count"
Progress bar
.stage-bar + .stage-bar-fill — 4px, amber-deep→amber-fill gradient, inline width:%

Light-mode notes
All amber uses --amber-text for copy (#946C00 — AA pass) and --amber-fill for strokes/fills.
No drop-shadow glow on active cells — uses --amber-wash tint + --sh-sm shadow instead.
04 · Using this for the manager hub
Recommended adaptations when you extend this:
  • For per-rep rows, stack several .stage-flow blocks with a rep label on the left — compact variant fits neatly.
  • Only one .active cell per row — it's the "where the rep is focused now" signal.
  • Keep cell copy tight: stage-name should be 1–2 words, never wrap. If it might wrap, shorten or add a title attribute.
  • Avoid colour-coding cells per rep — keep done / active / next as the only semantic states. Use a small colored dot in the rep label column if you need per-rep identity.
  • On hover, consider a subtle lift (transform: translateY(-1px)) + border darken to --border-dk. Don't change chamfer direction — it disturbs rhythm.
  • For a manager rollup, the progress-bar width should reflect weighted-forecast progress, not just cell count.
