01 · Brand posture

Conga is the brand users see. Znith is the studio that built it. Every tool in the suite is a Conga product — "Conga Sales Intelligence," "Conga Win Room," "Conga Account Brief." Znith is the invisible design system powering them, credited quietly in the footer as /Znith.

RuleHeader = Conga. Footer = /Znith. Amber = internal tool signature. Never mix.
  • Header uses the Conga Symbol SVG and the product string Conga [Product Name]. Never the word "Znith."
  • Footer carries the /Znith wordmark as a quiet build credit — the only place the Znith identity is visible in-product.
  • Amber (#E5A83A) is the internal-tool signature. It signals this is a product tool for sales teams, not a customer-facing Conga marketing surface.
  • Purple (Conga brand color) is reserved for Conga-branded moments only — login splash, marketing callouts, "Powered by Conga" credits. Never inside the tool chrome itself.

04 · Tokens · Light

Light theme is a parallel design, not a token flip. Amber retuned, shadows replace glows, solid hairlines replace translucent borders.

07 · Geometry (chamfer)

The defining signature. CSS clip-path polygon cuts the corners instead of border-radius.

--clip-lg: polygon(10px 0%, ...);   /* cards, panels, modals */
--clip-sm: polygon(5px 0%, ...);    /* buttons, chips, badges */
--clip-fwd: polygon(0 0, ...);       /* directional: forward (flow/progress) */
--clip-lead: polygon(12px 0, ...);    /* directional: leading-edge (alerts) */
Rule · ChamferDo not chamfer: form inputs (focus rings get clipped), dropdowns/tooltips (overflow:visible breaks), sticky headers (composite breaks). Use border-radius: 3px on these instead.

08 · Number treatment

Znith is a sales product. Numbers are the product. They get editorial care.

  • Always tabular: font-variant-numeric: tabular-nums; on every number.
  • Unit separation: hero numbers render unit at 40% size and 50% opacity — $24.8M
  • Delta chips: dedicated component for ±values — arrow glyph integrated, color by direction (green up, red down).
  • Letter-spacing: hero numbers use -0.04em.

09 · Amber states

Two intensities prevent amber fatigue.

StateUseTreatment
Default amberStatic accents, labels, bordersSolid --amber only
Charged amberLive / acting / changing states+ --sh-amber glow (dark) or shadow (light) + tick-mark edge
Rule · ChargedReserve for elements that are actually changing. Live pips, generating indicators, newly-arrived alerts, KPIs closing this week. If every amber element is charged, nothing pops.

11 · Data visualization

  • Line charts: 1.8px amber trace, dotted grid at 25/50/75%, leading dot on live endpoint, mono axis labels in --tghost.
  • Bar charts: solid amber fills, 1px divider at zero, tabular values above bars on hover.
  • Score bars (MEDDPICC): 8-segment bar graph, bright amber filled + deep amber partial + surface-3 empty.
  • Risk bands: 2σ band in relevant status color at 8% opacity + dashed stroke.
  • Never: 3D, gradient-as-data, donuts >3 segments, stacked bars without legend, emoji in charts.

12 · Empty / loading / error

Empty

Mono eyebrow (NO DATA · YET) + one line of Epilogue + one outline CTA. Dashed --border. No illustrations.

Loading

Amber 2px bar sweeping horizontally across the card bottom (charged-amber motion language). Mono readout SYNCING · 3.2s. No spinners.

Error

Directional chamfer pointing away (--clip-lead reversed), 2px risk-red stripe, mono error code (ERR · 4402), plain-language message, retry button.

13 · Accessibility

  • Faint text tier: --tfaint at 0.42 (was 0.28) passes WCAG AA on surface at 13px+.
  • Ghost tier: --tghost at 0.22 is decorative only — axis labels, deep-background chrome. Never used for text users must read.
  • Amber on dark: #E5A83A on #05070F → contrast 8.4:1. AAA pass.
  • Amber on light: #946C00 on white → contrast 5.2:1. AA pass for normal text.
  • Focus rings: 2px solid amber with 2px offset. Never rely on chamfer for focus affordance.

14 · Do not

  • Do not use the word "Znith" in user-facing UI outside the footer wordmark.
  • Do not use Conga purple inside tool chrome — reserved for marketing surfaces.
  • Do not use border-radius on cards/panels — use clip-path.
  • Do not chamfer form inputs, dropdowns, or sticky elements.
  • Do not use --tfaint below 13px or --tghost for readable text.
  • Do not invent new amber shades. Stay on the two-intensity ladder.
  • Do not add illustration or stock photography inside the tool — only uploaded logos/avatars.
  • Do not emoji. Use mono glyphs or SVG icons.
  • Do not stack more than one ambient pulse per screen.
  • Do not use the charged-amber treatment on static elements.
