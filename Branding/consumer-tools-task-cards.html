APPENDIX APR workflow

One tool = one PR = one fresh Claude Code session. Do not stack multiple cards in one session. The failure mode on the last sweep was context exhaustion — agent half-remembers work across files, drops steps.

Per PR:

  1. Start a fresh Claude Code session. Paste ONLY that tool's card (TC-01 through TC-07).
  2. Let the agent work through steps 1–N.
  3. Before merging, run the Acceptance Tests at the bottom of the card. All must output the expected value.
  4. If any test fails, reject the PR with the failing test line. Do not negotiate — the test is the contract.
  5. After merge, manually open the tool at data-theme="dark" and data-theme="light". Walk every screen.

APPENDIX CWhy this handoff shape works

Small scope per session — one card = 300 KB of HTML, one tool's mental model. Agent doesn't exhaust context.

Explicit tests — no ambiguity about "done." A grep count is a pass/fail, not a judgment call.

Diagnostic up front — each card shows what's wrong right now with check/fail marks. Agent doesn't have to infer priority.

No prose-heavy instructions — universal rules are listed once at the top; each card is edits + tests, nothing more.

Replay-safe — if a PR fails tests, the card is idempotent. Re-running produces the same result.
